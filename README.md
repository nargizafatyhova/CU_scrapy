# Краулер новостей KP.ru

> Проект по практике — сбор и визуализация новостного датасета с сайта «Комсомольская Правда»
>
> **Автор:** Фатыхова Наргиза, 2026

Асинхронный краулер на базе Scrapy, который обходит рубрики kp.ru, извлекает тексты и метаданные статей, кодирует изображения в Base64 и складывает всё в MongoDB. Для просмотра собранных данных поднимается FastAPI-дашборд с пагинацией и превью картинок.

## Стек

- **Python 3.10+** — основной язык
- **Scrapy** — асинхронный сбор данных
- **MongoDB** — хранилище документов
- **FastAPI** — веб-интерфейс для просмотра

## Быстрый старт

```bash
# 1. Установка зависимостей
source venv/bin/activate
pip install -r requirements.txt

# 2. Убедиться, что MongoDB запущена
brew services start mongodb-community@7.0

# 3. Запуск краулера
cd kp_project
scrapy crawl kp          # Ctrl+C для мягкой остановки

# 4. Запуск дашборда (http://127.0.0.1:8000)
fastapi dev dashboard.py
```

## Архитектура

**Краулер** стартует с 10 рубрик (политика, экономика, спорт и др.) и рекурсивно следует по ссылкам, фильтруя URL по паттернам `/daily/` и `/online/`. Из каждой статьи извлекаются заголовок, описание, текст, дата публикации, автор, ключевые слова и ссылка на главное фото.

**Пайплайны** обрабатывают каждый элемент последовательно:
1. `PhotoBase64Pipeline` — скачивает фото и кодирует в Base64
2. `MongoPipeline` — сохраняет готовый документ в коллекцию `articles`

**Дашборд** подключается к той же базе и отображает записи с пагинацией (50 на страницу), рендерит Base64-картинки и показывает превью текста в тёмной теме.

### Ключевые параметры краулера

| Параметр | Значение | Эффект |
|----------|----------|--------|
| `CONCURRENT_REQUESTS` | 32 | Параллельные запросы |
| `DOWNLOAD_DELAY` | 0 | Без задержек между запросами |
| `COOKIES_ENABLED` | False | Без cookie |
| `CLOSESPIDER_ITEMCOUNT` | 11 000 | Автостоп после N статей |
